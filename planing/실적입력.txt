현재 기능을 동일하게 유지하면서 프라임뷰로 만들어줘.
기능에 대해 다시 한번 정리하면,

항목은 총 11개(9개 + 삭제, 추가)이고
No, 처방월, 제품명, 보험코드, 단가, 처방수량, 처방액, 처방구분, 비고 + 삭제, 추가

9개 항목 중 이용자가 편집 가능한 항목은 5개
처방월, 제품명, 처방수량, 처방구분, 비고
따라서 이 5개들간에는 상하좌우 화살표로 자유롭게 이동이 가능해야 함

보험코드와 단가는 제품 선택시 자동으로 불러와짐
처방액은 불러와진 제품의 단가 x 입력한 수량으로 자동 계산됨

처방구분은 EDI가 디폴트값으로 들어가고 이용자가 변경할 수 있음
처방월도 정산월 - 1M이 디폴트로 들어가고 이용자가 변경할 수 있음
비고는 필수 항목이 아님

이용자가 주로 입력하는 항목은 제품명과 처방수량이므로 엔터로 이동이 가능해야 함

제품명 선택시 자동으로 같은 행의 처방수량으로 이동해야 함
한 행의 모든 값이 입력되었으면(비고를 제외) 처방수량에서 엔터 입력시 다음 빈행이 생기고 그 행으 제품명으로 이동해야 함.
제품명을 미입력하고 처방수량에서 엔터를 입력했을 경우는 제품명으로 이동해야 함 (지금 새로 추가한 정책)

선택 가능한 제품은 처방월(제품 관리 중 기본월)에 등록되고 활성화된 제품임

만일 제품을 먼저 선택한 상태에서 사후 처방월을 변경했을 경우 변경된 처방월에 미리 선택한 해당 제품이 없다면 선택되었던 제품과 입력했다면 입력한 처방수량값이 모두 없어저야 함.

비고를 제외한 모든 값이 다 있어야 1개의 행으로 간주함.

이 조건들을 잘 기억하고, 프라임뷰로 변경해줘.



실적 입력(Performance Register) 화면 규칙 정리

1. 행(로우) 구조 및 개수
정산월-거래처(병원)별로 입력 화면이 생성됨.
기본 100개 입력 행이 자동 생성됨(추가/삭제로 개수는 유동적).
합계행(맨 아래)은 항상 고정되어 자동으로 수량/처방액 합계를 보여줌.
중간에 빈 행이 있어도 무방하며, 사용자는 자유롭게 입력/삭제/추가 가능.

2. 컬럼(필드) 및 입력 방식

정산월, 거래처(병원):
이 화면에서는 수정 불가
이전 화면에서 선택된 값이 상단에 고정 표시됨.

처방월, 제품명, 보험코드, 약가, 수량, 처방액, 처방구분, 비고, 작업 순서로 컬럼이 나옴.

처방월, 제품명, 처방구분:
드롭다운으로 선택

제품은 보험코드 하나씩만

보험코드, 약가:
제품명 선택 시 자동 입력
직접 수정 불가(읽기전용)

수량:
숫자 입력, 입력 시 처방액 자동 계산

처방액:
약가×수량 자동 계산, 직접 입력 불가

비고:
자유 입력

작업:
각 행마다 추가/삭제 버튼이 있어, 중간에도 행 추가/삭제 가능

3. 행 추가/삭제
상단에 '행 추가' 버튼 없음
각 행의 '작업' 컬럼에서 '추가'/'삭제' 버튼으로만 조작
합계행은 항상 맨 아래 고정(추가/삭제 불가)

4. 입력/저장 규칙
입력 건수/저장은
모든 필수값(처방월, 제품명, 수량, 약가)이 입력된 행만 카운트/저장
빈 행/불완전 행은 저장/카운트에서 제외

5. UI/UX
PrimeVue DataTable 사용,
헤더와 합계행은 sticky(고정)
입력 행만 스크롤
입력 필드/드롭다운은 PrimeVue 컴포넌트로 통일
정산월/거래처 정보는 상단에 고정 표시

6. 데이터 연동
제품/처방월/처방구분 등은 Supabase에서 실시간 fetch
제품명 선택 시 보험코드/약가 자동 채움
처방월 드롭다운은 정산월 기준 최근 3개월
필요시 이 규칙을 문서로 저장하거나, 추가/수정 요청도 가능합니다!
