📋 전체 프로세스 정리
1단계: 기존 데이터 존재 확인
조건: 업로드 파일의 기준월에 기존 제품이 있는 경우
메시지: "기존 데이터가 있습니다. 계속 등록하시겠습니까?"
확인: 2단계로 진행
취소: 작업 중단

2단계: 추가 vs 대체 선택
메시지: "기존 데이터에 추가하시겠습니까? 대체하시겠습니까?"
확인: 추가 모드 → 3단계 (중복 체크)
취소: 대체 모드 → 기존 데이터 삭제 후 바로 등록 (중복 체크 건너뜀)

3단계: 중복 체크 (추가 모드에서만)
표준코드 중복 확인
중복 없음: 바로 등록
중복 있음: 4단계로 진행

4단계: 중복 발견 시 계속 진행 여부
메시지: 중복 에러 내용 + "계속 등록 작업을 진행하시겠습니까?"
확인: 5단계로 진행
취소: 작업 중단

5단계: 중복 해결 방법 선택
메시지: "이미 동일한 표준코드 제품을 어떻게 처리하시겠습니까?"
확인: 기존 제품 삭제 후 신규 제품 등록 (교체)
취소: 중복되지 않는 신규 제품만 등록 (기존 유지)
