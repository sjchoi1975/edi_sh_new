이제 관리자가 이용자가 등록한 파일을 확인하는 화면과 기능을 만들거야.
실적 관리 > 내역 조회에 만들건데
먼제 메뉴명을 내역 조회에서 등록 현황으로 바꾸고,
이용자가 등록 현황에서 확인하는 화면과 아주 거의 동일하게 만들면 돼.

차이점은 아래 2가지야.
1. 이용자는 본인이 등록한 내역만, 관리자는 모든 이용자가 등록한 (자기가 이후 추가할 관리자 등록 기능에서 등록한 내역까지 포함)해서 조회 가능
-> 테이블 위에 병원 선택 외에 업체 선택도 추가 필요

2. 이용자는 실적 입력 가능 기간이라면 수정도 가능하지만, 관리자는 조회만 가능.
-> 삭제, 추가 버튼 항목, 저장 버튼 관련 사항 제거

기본 UI, 디자인의 일관성을 위해
이용자용 PerformanceRegisterList.vue 파일을
관리자용에 AdminPerformanceRegisterList.vue 라고 이름을 바꿔서 넣었으니 참고해서 수정해줘.

수정 불가 셀에 연회색이 배경으로 들어가는 것에 대해 다시 정의할게.
등록하는 화면에서 등록 가능 셀과 선택, 수정 불가 셀을 구분하기 위해 배경색을 넣은거야.

그런데 등록 현황은 
이용자는 제출기간에 따라 제품명, 수량 등을 수정이 가능할수도 있고, 기간이 지났으면 수정을 못할수도 있어.
관리자는 모두 수정이 불가한거고.
따라서 등록 화면에 적용했던 방식을 적용하지 않고 아래 새로운 규칙을 적용할거야.

관리자는
등록 현황은 모두 수정 불가이므로, 이에 따라 이미 삭제, 추가, 저장 버튼도 다 제거했으므로,
굳이 수정 불가라고 셀 배경색을 연회색으로 할 필요 없음.
그냥 모두 일반 흰색으로 하면 됨

이용자는
선택한 정산월에 따라
1) 입력 기간이 아닌 정산월이면,
    위 관리자에 적용한거처럼 No. 거래처, 처방월 ~ 비고까지 배경색 변화 없이 모두 일반 흰색으로 하고,
    삭제, 추가 버튼 항목만 셀 배경색을 연회색으로 그리고 안에 있는 삭제, 추가 버튼을  (btn-delete-m-d, btn-add-m-d)  로 바꿔주면 돼.

2) 입력 기간인 정산월이면
실적 등록 화면과 동일해.

그런데 이제 보니 실적 등록 화면에 수정사항이 몇가지 있네.
그걸 먼저 해야겠다.

지금 실적 등록 화면에 테이블 내용셀이 모두 연회색으로 되어있어.
이용자가 선택, 수정이 불가한 항목만 연회색으로 하면 되는거야.
보험코드, 약가, 처방액 3가지야!
삭제, 추가 버튼 있는 셀도 이용자가 선택 가능이니 연회색 배경 하면 안돼!

그리고 병원 선택 방식도 바꿀건데.
우선 위의거 먼저 하자.
